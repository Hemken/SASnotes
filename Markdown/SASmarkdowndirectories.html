Setup

In this example, all files should be in the same directory. That is, the rmd file, the (usually temporary) md file, and the final document (in this case, this html page you are reading).

getwd() # Initial location within a knitr eval session
## [1] "Z:/PUBLIC_web/SASworkshops/Markdown"

This is also where knitr will write out temporary SAS code files (*.sas files). When SAS is called to evaluate these code files, this will be SAS's default location, per SAS's batch processing rules.

saspath <- "C:/Program files/SASHome/SASFoundation/9.4/sas.exe"
sasopts <- "-nosplash -ls 75"
knitr::opts_chunk$set(engine='sas', engine.path=saspath, engine.opts=sasopts, comment="")

Although the sasopts (SAS command line options when calling SAS in batch mode) might include directory locations for log and listing output, here we have specified none.

Both knitr and SAS give you options for directing files to other locations, just remember that these will need to be set separately. Within knitr, the options are package options named named root.dir and base.dir.
